## Вступ

Цей документ пояснює теоретичні та практичні основи прикладу `geometry_triangles.ml` — четвертого великого тематичного прикладу minilog після `family` (родинні зв'язки), `biology` (таксономія) та `dwh_dependencies` (залежності даних).

Приклад демонструє, що арифметичні можливості Phase 9 (оператори `+`, `-`, `*`, `/` та функції `sqrt`, `pow`) дозволяють minilog виконувати реальне числове обчислення всередині логічних правил — конкретно, класифікувати трикутники за сторонами і кутами, перевіряти нерівність трикутника та верифікувати теорему Піфагора.

**Для кого:** Для тих, хто вже пройшов приклади Phase 1 і хоче бачити, як логічне програмування працює з числовими обчисленнями.

**Пререквізити:** Phase 9 арифметика (MINILOG-021..024, MINILOG-061). Див. розділ "Арифметика" в [docs/language-reference.md](language-reference.md).

## Геометрія як формальна система

Геометрія — одна з перших формальних систем в історії. Евклід (~300 р. до н.е.) побудував її з п'яти аксіом, загальних понять та визначень, з яких виводив теореми через ланцюги логічних кроків.

Цей підхід природно відображається на Horn clauses:
- **Аксіоми** стають фактами і правилами
- **Теореми** стають запитами
- **Доведення** — це proof trees, які minilog будує автоматично

```
% Факт (аксіома): точка має координати
точка(a, 0, 0).

% Правило (визначення): як обчислити відстань
правило відстань(?p1, ?p2, ?d) якщо ...

% Запит (теорема): чи є трикутник піфагоровим?
?- піфагоровий(t1).
```

Гільберт (1899) формалізував геометрію строго; Ковальскі (1974) показав, що будь-яка формальна система може бути виражена як логічна програма. `geometry_triangles.ml` — конкретна реалізація цієї ідеї: елементарна планіметрія в ~100 рядках minilog.

## Що моделює приклад

### Точки

Точка — це факт `точка(назва, x, y)`:

```
точка(a, 0, 0).
точка(b, 4, 0).
точка(c, 0, 3).
```

Координати задані як константи. minilog не обчислює координати — він класифікує та верифікує відношення між заданими точками.

### Трикутники

Трикутник — факт `трикутник(назва, точка1, точка2, точка3)`:

```
трикутник(t1, a, b, c).   % прямокутний 3-4-5
```

### Кути

Кути задані як факти, бо minilog не має тригонометрії (`sin`, `cos`, `atan2`):

```
кут(t1, a, 90).    % прямий кут у вершині a
кут(t1, b, 37).
кут(t1, c, 53).
```

Якщо б minilog мав тригонометрію, кути можна було б обчислювати з координат.

## Обчислення довжин сторін

Формула евклідової відстані:

$$d = \sqrt{(x_2 - x_1)^2 + (y_2 - y_1)^2}$$

У minilog — одне правило з `is/2` binding:

```
правило відстань(?p1, ?p2, ?d) якщо
    точка(?p1, ?x1, ?y1) і точка(?p2, ?x2, ?y2) і ?p1 != ?p2 і
    ?d = sqrt((?x2 - ?x1) * (?x2 - ?x1) + (?y2 - ?y1) * (?y2 - ?y1)).
```

Тут `?d = sqrt(...)` — це `is/2` binding (MINILOG-061): правий бік обчислюється арифметично, результат зв'язується зі змінною `?d`.

Для трикутника t1 (a=(0,0), b=(4,0), c=(0,3)):
- AB = sqrt((4-0)^2 + (0-0)^2) = sqrt(16) = 4.0
- BC = sqrt((0-4)^2 + (3-0)^2) = sqrt(25) = 5.0
- AC = sqrt((0-0)^2 + (3-0)^2) = sqrt(9) = 3.0

Правило `сторони/4` обчислює всі три сторони одним запитом:

```
правило сторони(?t, ?ab, ?bc, ?ac) якщо
    трикутник(?t, ?a, ?b, ?c) і
    відстань(?a, ?b, ?ab) і відстань(?b, ?c, ?bc) і відстань(?a, ?c, ?ac).
```

## Класифікація за сторонами

### Рівнобедрений

Трикутник рівнобедрений, якщо принаймні дві сторони рівні. Три правила покривають усі пари:

```
правило рівнобедрений(?t) якщо сторони(?t, ?a, ?b, ?c) і ?a = ?b.
правило рівнобедрений(?t) якщо сторони(?t, ?a, ?b, ?c) і ?b = ?c.
правило рівнобедрений(?t) якщо сторони(?t, ?a, ?b, ?c) і ?a = ?c.
```

### Рівносторонній

Усі три сторони рівні. Рівносторонній трикутник є підмножиною рівнобедрених:

```
правило рівносторонній(?t) якщо сторони(?t, ?a, ?b, ?c) і ?a = ?b і ?b = ?c.
```

### Різносторонній

Всі три сторони різні — виражається через заперечення рівнобедреності.

## Класифікація за кутами

### Прямокутний

Один кут = 90 градусів:

```
правило прямокутний(?t) якщо кут(?t, ?v, 90).
```

### Тупокутний

Один кут > 90 градусів:

```
правило тупокутний(?t) якщо кут(?t, ?v, ?a) і ?a > 90.
```

### Гострокутний

Жоден кут не прямий і не тупий — виражається через negation-as-failure:

```
правило гострокутний(?t) якщо трикутник(?t, ?p1, ?p2, ?p3) і не прямокутний(?t) і не тупокутний(?t).
```

Це той самий патерн, що і `літає(?x) якщо є(?x, птах) і не не_літає(?x)` з `biology.ml` — negation-as-failure для класифікації "за виключенням".

## Теореми

### Нерівність трикутника

Для будь-якого трикутника з сторонами a, b, c: a+b > c, b+c > a, a+c > b.

```
правило валідний_трикутник(?t) якщо
    сторони(?t, ?a, ?b, ?c) і
    ?a + ?b > ?c і ?b + ?c > ?a і ?a + ?c > ?b.
```

Для t5 (колінеарні точки: a=(0,0), b=(4,0), g=(8,0)) сторони 4, 4, 8:
- 4 + 4 > 8 → false (4 + 4 = 8, не строго більше)

Тому `валідний_трикутник(t5)` повертає `false` — це вироджений випадок.

### Теорема Піфагора

Для прямокутного трикутника: a^2 + b^2 = c^2 (де c — гіпотенуза).

```
правило піфагоровий(?t) якщо сторони(?t, ?a, ?b, ?c) і ?a * ?a + ?b * ?b = ?c * ?c.
```

Три правила покривають усі можливі розташування гіпотенузи (яка з трьох сторін найдовша). Для t1: 4^2 + 3^2 = 16 + 9 = 25 = 5^2 = 25.

Proof tree для `?- слід піфагоровий(t1).` показує повний ланцюг:
1. Знайти трикутник t1 (a, b, c)
2. Обчислити три відстані через `sqrt` і `pow`
3. Перевірити арифметичну рівність

## Сильні сторони та обмеження

### Сильні сторони

- **Декларативність:** правила класифікації — це прямий переклад геометричних визначень
- **Proof trees:** кожна відповідь має повний ланцюг доведення, який можна перевірити
- **Композиція:** арифметика Phase 9 природно вбудовується в правила через `is/2` binding
- **Ієрархія:** рівносторонній ⊂ рівнобедрений — показується через окремі правила

### Обмеження

- **Немає тригонометрії:** кути задаються як факти, не обчислюються. Для обчислення потрібні `sin`, `cos`, `atan2`
- **Немає кіл:** тільки точки, відрізки і трикутники
- **Немає перетинів:** minilog не знаходить точки перетину прямих чи кіл
- **Немає числового розв'язувача:** minilog не може знайти координати точки, що задовольняє систему рівнянь
- **Float порівняння:** `sqrt` повертає float, тому `=` порівнює з плаваючою комою. Для точних обчислень потрібна раціональна арифметика

## Зв'язок з ширшим контекстом

Цей приклад — конкретна реалізація ідеї "формальні системи як reasoners над обчисленими величинами". Геометрія була однією з перших формальних систем (Евклід, ~300 р. до н.е.), і механізація елементарної класифікації трикутників у ~100 рядках minilog демонструє традицію від Евкліда через Гільберта до Ковальскі та сучасного логічного програмування.

Для подальшого вивчення:
- **Sterling & Shapiro**, "The Art of Prolog" — класичний підручник
- **Clocksin & Mellish**, "Programming in Prolog" — практичний вступ
- **Bratko**, "Prolog Programming for AI" — геометричні приклади в розділі про constraint solving

Повна тригонометрія та евклідові конструкції виходять за межі Phase 10, але можуть з'явитися в наступних етапах, якщо minilog додасть `sin`, `cos`, `atan2` як built-in функції.
